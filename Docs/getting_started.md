## 2. Python Setup

The Python layer acts as the "Heavy Lifter" for the Cine Registry. It handles metadata enrichment via the TMDB API and provides a secure, sanitized interface for the PostgreSQL procedures.

### Install Dependencies

```bash
pip install -r requirements.txt
```

### Environment Variables

Create a **.env** file in the root directory.  
> Note: This file is included in .gitignore to ensure your private API keys and database credentials are never leaked to GitHub.

```env
TMDB_API_KEY=your_key
DB_NAME=focus_destroyer
DB_SCHEMA=the_receipts_sandbox
DB_USER=your_user
DB_PASSWORD=your_password
DB_HOST=localhost
DB_PORT=5432
```

### Usage

Run the logging utilities to fetch metadata and push records directly to the database via the command line:

- `python movie_logger.py`: Searches TMDB for a movie, fetches the tmdb_id, genres, and runtime, then calls the `movie_watch` procedure.

- `python add_series.py`: Initializes a new show in the registry. It pulls the total season count and official series data before calling `add_series` procedure.

---

### 3. API Key Acquisition

The Python scripts (`movie_logger.py, add_series.py`) act as the bridge between your database and the global film library. They require access to The Movie Database (TMDB) API to fetch poster paths, runtimes, and genres automatically.

1. **Sign Up:** Create a free account at [themoviedb.org](https://www.themoviedb.org/).
2. **Request API Key:** Navigate to your account settings > API to generate a new **API Read Access Token** or **API Key**.
3. **Local Configuration:** Paste this key into your `.env` file under `TMDB_API_KEY`.

> **Security Note:**  
Your API key is your digital signature. Never share it. The .env file is strictly for your local environment and is explicitly excluded from version control via .gitignore to prevent accidental leaks.
